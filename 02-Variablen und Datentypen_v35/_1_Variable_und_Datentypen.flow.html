Kommentare schreiben

Um Quellcode zu beschreiben, werden Kommentare verwendet.

Das ist immer wichtig und sinnvoll, da Sie immer wissen sollten, was Ihr Code tun soll.

1) Sie schreiben auf, was der Code tun SOLL - in einem Kommentar

2) Sie schreiben den Code, das IST das Programm, das compiliert und ausgeführt wird.

3) Wenn der Code nicht tut, was er soll, können Sie im Kommentar
    immer nachlesen, was er eigentlich tun sollte.
    D.h. Sie können das IST gegen das SOLL prüfen

Das Schreiben von Kommentaren entspricht den Schritten  Ü&A in der allgemeinen Vorgehensweise, die Sie in der ersten Aufgabe in der letzten Woche kennengelernt haben:

Lesen -> Verstehen -> Überlegen -> Aufschreiben -> Codieren

Wenn Sie im Code keine oder zu wenig eigene Kommentare schreiben, können Sie schon nach kurzer Zeit selber nicht mehr nachvollziehen, was Ihr eigener Code denn eigentlich tun soll, und auch nicht, wie er das tut. Das sollten Sie aber können, wenn Fehler auftreten, wenn der Code weiterentwickelt werden soll, wenn jemand anders mit ihm arbeiten soll, wenn Sie sich auf eine Prüfung vorbereiten, ...

Im Lösungs-Code sind die beiden Möglichkeiten aufgeführt, Kommentare zu schreiben:

  • Der Zeilen-Kommentar, der mit // beginnt und bis zum Ende der Zeile geht
  • Der Block-Kommentar, der mit /* beginnt, über mehrere Zeilen gehen kann und mit */ endet

Modulo

Ein eher selten verwendeter, aber dann sehr nützlicher, Operator ist der Modulo-Operator %, der den Rest einer ganzzahligen Division angibt.
Beispiel: 9 % 4 ist 1

Beispiel: Eine Fließkommazahl soll auf den nächsten Hunderter abgerundet werden

Beispiele: Ein Originalwert von 523.34 ergibt abgerundet 500.00, ein originalwert von 4567.39 ergibt abgerundet 4500.00

Der "Abstand" (Differenz) vom Originalwert zum gewünschten, abgerundeten Wert entspricht also genau der Berechnung von Originalwert modulo 100. Im ersten Beispiel ist die Differenz also 23,34.

Den abgerundeten Wert erhalten Sie daher, wenn Sie vom Originalwert die Differenz abziehen, im ersten Beispiel also: 523,34 - 23,34 = 500.

In Ihrem Lösungscode ist ein Ansatz zu einem kleinen Programm bereits vorgegeben, das diesen Rechenweg umsetzt. (Ja, das geht alles viel schneller und mit weniger Code, hier geht es darum, sowohl LVÜAC als auch den Umgang mit Variablen und dem Modulo-Operator zu üben).

double abgerundet ;

// für die Differenz zwischen orginalwert und abgerundetem Wert
double differenz ;

//   und dieser Variablen zugewiesen
