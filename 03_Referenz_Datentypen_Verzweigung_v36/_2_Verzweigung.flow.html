Verzweigungen

Sie führen nun Code aus, der nicht jedesmal das gleiche macht, sondern - je nach den abgefragten Bedingungen - unterschiedliche Aktionen ausführt.

Das wird erreicht durch Verzweigungen oder Fallunterscheidungen, wie Sie im Folgenden ausprobieren.

Für das aktive Lesen des Texts empfiehlt es sich, die Textmarker zu nutzen.
 
 
 

Die Umsetzung machen Sie der Reihe nach.

  • nur den Teil des Codes ausführen, der zu dieser Aufgabe gehört.
    • Wie das geht, wurde in der vorangegangenen Aufgabe nochmals erklärt.

Fehlermöglichkeiten bei Verzweigungen

Auch bei so einfachen Kontrollstrukturen wie Verzweigungen kann man etliche Fehler machen. Damit Sie diese Fehler kennen, erkennen und sie beheben können, probieren wir die wichtigsten nun aus. Sie können Sie dann erkennen und beheben, wenn sie Ihnen selber unterlaufen. 

 

Fehlerfall: Ein ; an der falschen Stelle

Ein typischer Fehler ist das Setzen eines Semikolons an einer falschen Stelle. Setzen Sie als gewollten Fehler ein ";" hinter die Klammer nach der Bedingung eines if-statements, z.B. so:

if(2>1);

    System.out.println("bedingung ist erfüllt");

Ergänzen Sie den Code im Code-Rahmen, wie schon kennengelernt als Fehlerfall.

Führen Sie dann nur (!) den fehlerhaften Code im Fehlerfall-Block aus.

Verschachtelte Verzweigung

Nun wollen wir nicht nur eine einfache Verzweigung sondern mehrere, verschachtelte Verzweigungen anwenden.

Anforderung:

Wenn die Höchstgeschwindigkeit überschritten wird, gibt es verschiedene Stufen der Sanktionen, die sich nach der Höhe der Differenz ergibt:

  1. Wenn die Differenz kleiner oder gleich null ist, gibt es natürlich keine Strafe
  2. sonst: Wenn die Differenz kleiner als 10 km/h ist, ist die Strafe 10 Euro
  3. sonst: Wenn die Differenz kleiner als 50 km/h ist, ist die Strafe 300 Euro, 1 Strafpunkt
  4. sonst: Wenn die Differenz kleiner als 70 km/h ist, ist die Strafe 500 Euro, 3 Strafpunkte und ein Führerscheinentzug auf 1 Jahr
  5. Ansonsten ist der Führerschein für immer weg

Geben Sie jeweilige Strafe als Text auf die Konsole aus. Das Muster dafür ist: „Sie sind xxx km/h zu schnell gefahren. Die Strafe dafür ist yyy“.

Lösungsweg:

Für den Anzeigewert eines Tachometers und für die erlaubte Höchstgeschwindigkeit hatten Sie bereits oben passend benannte Variablen eines passenden Datentyps definiert, und initialisiert.

Definieren Sie nun noch eine Variable für die Differenz zwischen gefahrener und erlaubter Geschwindigkeit, berechnen Sie ihren Wert und speichern Sie ihn dort. 

Zur Ermittlung des nun neu anzuzeigenden Texts

  • bauen Sie zuerst schrittweise einen String passend zusammen, mit einer verschachtelten Fallunterscheidung,
  • den Sie abschließend, d.h. nach der verschachtelten Fallunterscheidung, auf die Konsole ausgeben.

Die Schritte werden im Folgenden beschrieben. 

Wie Sie einen String zusammenbauen, haben Sie bereits gelernt.
 

 

Vorgehen:

Den Wert des abschließend auszugebenden Strings ermitteln Sie nun dadurch, dass Sie Verzweigungen "verschachteln", d.h. in einem Zweig einer Verzweigung eine weitere Verzweigung einbauen, in der jeweils die Höhe der Differenz geprüft wird:

- Falls die Differenz der ersten Stufe entspricht, bauen Sie den passenden Text zusammen.

- Andernfalls testen Sie, ob die Differenz der zweiten Stufe entspricht und so weiter.

Sie schreiben also Verzweigungen innerhalb von Verzweigungen. Dabei müssen Sie beachten:

  • Folgen Sie der o.a. Reihenfolge 1..5 für das Prüfen der Höhe der Differenz.
  • Rücken Sie den Code bei der Verschachtelung so ein, dass Sie den Überblick behalten können, um den Code verstehen und so formal und logisch korrekt schreiben können.
    • Diese Einrückung ist entscheidend dafür, dass Sie Ihren eigenen Code verstehen können - und natürlich auch dafür, dass andere das können!
  • Beschreiben Sie bei jedem Schritt die Anforderung in einem //-Kommentar.
    • Die jeweilige Anforderung können Sie aus der Aufgabe oben in Ihren Code kopieren. 

Den zusammengebauten String geben Sie erst abschließend, am Schluss, nach der verschachtelten Verzweigung auf die Konsole aus.

Implementieren Sie den gesamten Lösungsweg, indem Sie den entsprechenden Code im Code-Rahmen ergänzen. 

Testen Sie diese verschachtelte Fallunterscheidung mit unterschiedlichen (erlaubten und nicht erlaubten) Werten für die oben definierten Variablen. Dazu tragen Sie in den Code unterschiedliche Werte ein und führen den gesamten Code-Rahmen aus.

Mehrfache Fallunterscheidung

Wenn man nicht nur eine wahr/falsch- bzw. true/false-Bedingung - wie bisher - auswerten möchte, sondern eine mehrfache Fallunterscheidung mit verschiedenen Werten vornehmen möchte, dann kann die Verwendung von mehrfach verschachtelten if-else-Konstrukten unübersichtlich werden.

Daher gibt es auch in Java hierfür die Kontrollstruktur switch-case, die Sie nun einsetzen sollen.

Auswertung von einzelnen Werten

Bei interaktiven Anwendungen werden häufig Eingaben vom Anwender angefordert, je nach Eingabe passieren dann unterschiedliche Aktionen.

Da wir noch nicht mit Eingaben umgehen können, gehen wir wie folgt vor:

  • Definieren Sie eine Variable, in der der Buchstaben der Eingabe gespeichert wird
    • verwenden Sie einen passenden Datentypen
    • und geben Sie einen beschreibenden Namen
    • wobei Sie die Konventionen der Namensgebung für Variablen in Java beachten
  • Je nach Inhalt dieser Eingabe-Variablen geben Sie unterschiedliche Texte auf die Konsole aus.
    • bei einem q oder Q: "Programm wird beendet"
    • bei einem s oder S: "Daten werden gespeichert"
    • bei einem r oder R: "Daten werden berechnet"
    • bei allen anderen Eingaben: "Nur q,Q, s, S, r und R sind erlaubt".

Implementieren Sie diese mehrfache Fallunterscheidung im Code-Rahmen

 

Testen Sie diese mehrfache Fallunterscheidung mit unterschiedlichen, erlaubten und nicht erlaubten Werten für die Eingabe. Dazu

  • tragen Sie nacheinander unterschiedliche Werte in die Eingabe-Variable ein,
  • führen den Codeblock aus,
  • und prüfen, ob das erwartete Ergebnis angezeigt wird.

Auswertung von Strings in einer Mehrfach-Fallunterscheidung

In Java kann man nicht nur eine mehrfache Fallunterscheidung nach einem Zahlen- oder Buchstabenwert treffen, sondern auch nach Werten einer String-Variablen. Das ist nützlich, wenn man je nach Datei-Endung verschiedene Programme aufrufen möchte, um diese Dateien zu bearbeiten.

Bauen Sie eine mehrfache Fallunterscheidung auf, die unterschiedliche Texte auf die Konsole ausgibt, je nach der Endung, die natürlich in einer Variable steht.

  • bei .jpg geben Sie aus "Bilddatei wird geöffnet"
  • bei .doc geben Sie aus "Textdatei wird geöffnet"
  • bei .xls geben Sie aus "Spreadsheetdatei wird geöffnet"
  • bei anderen Endungen geben Sie aus "Nur .jpg, .doc und .xls sind erlaubt"

Testen Sie diese Fallunterscheidung mit unterschiedlichen, erlaubten und nicht erlaubten Werten für die Endung.
