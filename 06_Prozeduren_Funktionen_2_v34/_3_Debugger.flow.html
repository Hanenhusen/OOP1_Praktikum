Optional: Debugger

Erste Schritte mit dem Debugger

Um den Ablauf der Anweisungen in einem Programm Schritt für Schritt nachvollziehen zu können, ist in 5Code ein "Debugger" eingebaut. Den können Sie nun kennenlernen, müssen es aber nicht. Die schrittweise Verfolgung eines Programms mit dem Debugger kann aber Ihr Verständnis des Ablauf sehr unterstützen. 

  • Der Debugger in 5Code zeigt nacheinander alle auszuführenden Schritte (Zeilen) an.
  • Erst auf Klick wird der nächste Schritt ausgeführt. 
  • Zu jedem Schritt können die Werte aller gültigen Variablen angezeigt werden
    • Das gilt auch für jeden vergangenen Schritt!
Wer Debugger aus anderen IDEs kennt, merkt, dass das in 5Code etwas anders funktioniert.
 

Beim Öffnen dieses Projekts wurde der Debugger für Sie als Feature freigeschaltet. 

 

Vor dem Debuggen wird automatisch compiliert

Da auch beim Debuggen Code ausgeführt werden soll, muss er vorher compiliert werden. Wenn dabei Fehler auftreten sollten, müssen Sie die beheben, bevor Sie debuggen können.

Aktuelle Zeile und Zeilen-Hintergrund

Die nächste auszuführende Zeile steht immer ganz unten. 
Sie wird daher nicht besonders hervorgehoben, um nicht zu sehr zu verwirren.

Um mehrere Zeilen und ihre Zusatz-Informationen (s.u.) klar voneinander trennen zu können, werden die Zeilen abwechselnd mit einem Standard- und einem leicht farbigen Hintergrund angezeigt:

Quell-Code nicht ändern während des Debuggens

Da der Debugger auf die Zeilen des Quellcodes zugreift, wie sie beim Start existierten, sollten Sie den Quellcode nicht ändern während des Debuggens!

Wenn Sie das machen, werden zwar die neuen Zeilen im Debugger angezeigt, es werden aber die originalen Zeilen ausgeführt - die hat ja der Compiler für die Ausführung compilert (s.o.)! Außerdem kann es zu Verschiebungen von Zeilen kommen, was Sie ebenfalls in die Irre führt.  

 

Informationen zu jedem Schritt anzeigen

In jedem Schritt kann man sich zusätzliche Informationen anzeigen lassen. 

Klicken Sie dazu mit der Maus auf die Zeilennummer, dann wird über der Zeile folgendes angezeigt:

  • Von welcher Art ist der nächste Schritt?
    • z.B. Variablen-Definition, Zuweisung, Methoden-Aufruf, ...
  • Welche Variablen sind vor (!) der Ausführung dieses Schritts bekannt, und welchen Wert haben Sie?
    • Die Variable int punkte = 78 haben Sie selber definiert.
    • Die Angabe String[] args -> {} bedeutet: Das ist eine Variable (hier: ein Parameter der Prozedur main()) mit dem Namen args, vom Datentyp String-Array, sie verweist auf ein leeres Array.

Wichtig: Die angezeigten Werte gelten vor (!) der Ausführung der Zeile.

Wenn in der aktuellen Zeile Werte geändert werden, werden die geänderten Werte in den Informationen des nächsten Schritts angezeigt - nachdem die aktuelle Zeile ausgeführt wurde:

Hier wurde durch die Zuweisung punkte = 82; der Wert der Variablen punkte gändert.

Informationen zu jedem vergangenen Schritt anzeigen

Eine wichtige Hilfe für das Debugging, d.h. um einen Fehler zu finden, ist es, die Werte von Variablen vor den vorherigen Schritten zu kennen. In aktuellen IDEs müssen Sie sich diese Werte merken oder notieren. 

5Code merkt sich die Werte für Sie und stellt sie für den jeweiligen Schritt dar. Sie brauchen einfach nur die Infos zu jeder gewünschten Zeile aufzuklappen:

  • nach der Ausführung der Zeile 12, d.h. vor der Ausführung der Zeile 13, die Variable punkte den Wert 82 hat. 

 

Darstellung von Kontrollstrukturen 

Kontrollstrukturen werden besonders dargestellt, um sie beim Ablauf auch gut identifizieren zu können. Außerdem kann der Ablauf so besonders gut nachvollzogen werden.

Schleifen

Schleifen werden ebenfalls mit einem passenden Icon und einer Erklärung angezeigt, z.B.:

Falls Sie die Schleifen-Variable i in der Liste der Variablen vermissen:
Die Schleifen-Variable i ist erst nach der ersten Ausführung der Zeile gültig. Daher steht sie hier, vor (!) der Ausführung noch nicht in der Liste.

Nachdem diese Zeile ausgeführt wurde, werden die Anweisungen im Körper der Schleife ausgeführt. Auch hier werden die Schritte im Körper eingerückt. 

Nachdem die letzte Zeile des Schleifen-Körpers ausgeführt wurde, wird wieder die erste Zeile der Schleife angezeigt, da nun erst die Weitermach-Aktion ausgeführt wird und dann die Weitermach-Bedingung geprüft wird:

Zwischen der letzten Zeile des Schleifen-Körpers und der ersten Zeile der Schleife wird eine gestrichelte Linie angezeigt. Diese deutet an, dass der normale Kontrollfluss von oben nach unten nicht mehr einfach zur nächsten Zeile übergeht, sondern zurück gesprungen wird. 

Diese Erläuterung kann man auch durch Aufklappen dieser Zeile angezeigt bekommen:

Nun kann man Schritt für Schritt verfolgen, wie z.B. die Zähl-Variable i hochgezählt wird. 

Erinnerung: die Werte von i in den vorangegangenen Schritten oder Durchläufen der Schleife können Sie immer auch nachträglich anzeigen - einfach durch Aufklappen der jeweiligen Zeile. Das ist sehr hilfreich für das Verständnis. Denken Sie dabei immer nur daran, dass die Werte vor (!) der Ausführung der Zeile dargestellt werden.

Methode wird beendet

Wenn Sie alle Schritte der Methode ausgeführt haben, kehrt der Kontrollfluss wieder zurück von dieser Methode (zeicheLinie()) zur aufrufenden Methode (main()). Das wird wieder mit einer gestrichelten Linie im Kontrollfluss des Debuggers dargestellt:

Methode zu Ende ausführen lassen und zur aufrufenden Methode zurückkehren

Falls Sie beim Debuggen feststellen, dass der Rest der Methode ausgeführt werden soll, ohne dass Sie den restlichen Code der Methode schrittweise ausführen müssen, können Sie das tun:

Ein Klick auf das Icon mit dem Pfeil nach oben macht genau das:

  • Der restliche Code der Methode wird ausgeführt
    • Bis zum Ende der Methode
  • Der Kontrollfluss springt zurück zur aufrufenden Methode
  • Der Debugger bleibt da stehen und zeigt den nächsten Schritt nach dem Aufruf an. 

 

Das Programm ist durchgelaufen bzw. beendet

Wenn die Methode main() komplett durchgelaufen ist, ist das gesamte Programm beendet. Dann können Sie natürlich keine weiteren Schritte ausführen. Daher sind dann die Steuer-Icons ausgegraut:

Um nicht immer alle Informationen gleichzeitig angezeigt zu bekommen, können Sie die Menge an Informationen über ein paar Icons steuern und einstellen. Dazu finden Sie am rechten Rand des Debug-Fensters ein paar Icons.
