Funktionen

Unterschied zu Prozeduren

Funktionen sind sehr ähnlich wie Prozeduren,

  • In ihnen passiert etwas
  • Plus: Sie haben auch einen Wert als Ergebnis

Auch für Funktionen gilt:

  • Definition - es wird einmal beschrieben, was in der Prozedur passieren soll
  • Aufruf - es kann beliebig oft bestimmt werden, dass es passieren soll

Auch hier gilt: Definition und Aufruf müssen an unterschiedlichen Stellen im Code positioniert werden, wie Sie bereits in der Aufgabe zu Beginn gelernt haben.

Neben der reinen Funktionalität und der korrekten Syntax müssen Sie die weiteren Randbedingungen beachten, die im Coding Style vorgegeben sind. Für Funktionen gilt, sehr ähnlich wie für Prozeduren:

- Geben Sie jeder Funktion und ihren Parametern gut beschreibende Namen. Der Name einer Funktion sollte den Rückgabewert beschreiben. 

- Zu jeder Definition einer Funktion gehört auch immer die Beschreibung, WAS diese Prozedur tun soll. Das geschieht im javadoc-Format, in dem auch die Parameter und der Rückgabewert mit speziellen Tags beschrieben werden. Die Beschreibung kann kurz sein, muss aber vollständig sein.

 

Satz des Pythagoras

Im Folgenden erstellen Sie einfache Funktionen um ihre Eigenschaften kennenzulernen. Auch hier schreiben und verwenden Sie mehrere Funktionen, um den Umgang mit Definition und Aufruf zu üben.

Als Beispiel verwenden Sie den einfachen Satz des Pythagoras.

   

 

Quadrat berechnen

Definieren Sie eine einfache Funktion, die das Quadrat einer übergebenen ganzen Zahl berechnet - und das Ergebnis der Berechnung als Funktionswert zurückgibt.

Vorsicht: Die Schreibweise a^2 bedeutet hier im Text a hoch 2. ABER: In Java bedeutet diese Operation keine Quadrierung!
 

Testen Sie diese Funktion, indem Sie sie zweimal, mit unterschiedlichen Werten aufrufen und das Ergebnis auf die Konsole ausgeben.

Erinnerung: "Als Funktionswert zurückgeben" ist etwas ganz anderes, als "Auf die Konsole ausgeben"
Überlegen Sie, welchen Datentyp die Funktion hat.
 

 

Summe von Quadraten berechnen 

Nun soll das Quadrat von zwei Zahlen berechnet und die Summe der Quadrate als Funktionswert zurückgegeben werden.

Definieren Sie hierfür eine passende Funktion, der diese zwei Zahlen übergeben werden. 

Testen Sie diese Funktion, indem Sie sie mit zwei Werten aufrufen und das Ergebnis auf die Konsole ausgeben.

In dieser Funktion rufen Sie die eben definierte einfache Funktion zur Berechnung eines Quadrats auf.
 

 

Mit Pythagoras prüfen

Der Satz des Pythagoras (a^2 + b^2 = c^2) gilt nur für rechtwinklige Dreiecke. Durch Vergleich von c^2 mit a^2 + b^2 kann man also feststellen, ob ein Dreieck mit den Katheten-Längen a und b und der Hypothenusen-Länge c ein rechtwinkliges Dreieck ist. 

Definieren Sie eine Funktion, die a, b und c als Parameter übergeben bekommt und die gerade beschriebene Prüfung durchführt. Die Funktion gibt also zurück, ob a^2 + b^2 = c^2 wahr ist. 

Testen Sie diese Funktion, indem Sie sie mit passenden Werten aufrufen und das Ergebnis auf die Konsole ausgeben. (3,4,5 sollten true ergeben, 1,2,3 sollte false ergeben).

Alles zusammenfassen

Zum Schluss fassen wir die oben geschriebenen Funktionen in einer Prozedur, zusammen. Dieser werden a, b und c als Parameter übergeben, sie prüft, ob das damit beschriebene Dreieck ein rechtwinkliges Dreieck ist und gibt dann eine dazu passende Meldung auf die Konsole aus (z.B. "Das Dreieck mit einer Hypothenuse der Länge 5 und zwei Katheten der Länge 3 und 4 ist rechtwinklig"). 

Definieren Sie  diese Prozedur.

Testen Sie diese Prozedur indem Sie sie mit passenden Werten aufrufen (z.B. 3,4,5 und 1,2,3) Wird der erwartete Text auf die Konsole ausgegeben?

Machen Sie sich klar, dass diese Prozedur keinen Funktionswert zurückgibt, sondern in ihrem Code etwas berechnet - mit Hilfe der oben definierten Funktionen - und dann etwas auf die Konsole ausgibt. 

 

Fehlerfälle testen

Bei Definition und Aufruf von Funktionen können die gleichen Fehler auftreten wie bei Prozeduren. Zusätzlich können Fehler bezüglich des Funktions-Datentyps (d.h. des Datentyps, den die Funktion zurückgibt)  auftreten. Einige typische Fehlerfälle sollten Sie kennen, damit Sie diese Fehler erkennen und beheben können.
