Programmieren in 5Code

Programme schrittweise erstellen 

Wie bereits besprochen, versteht man unter Programmieren man den kompletten Weg vom Problem zum Programm.

Es kommt praktisch nie vor, dass man seinen Code spontan und sofort nach dem Lesen der Anforderungen erstellt. Praktisch immer macht man das in mehreren Schritten - weil man schrittweise Anforderungen liest, versteht, Lösungen überlegt, notiert und dann codiert. 

Damit ist auch schon fast das ganze, sehr komplexe Gebiet des Software-Engineerings beschrieben. 

Die dort definierten Phasen Analyse, Design, Implementierung lassen sich leichter verständlich auch so benennen:

Lesen -> Verstehen -> Überlegen -> Aufschreiben -> Codieren

oder auf Englisch:

Read it -> Get it -> Think it -> Note it -> Code it

Diese Vorgehensweise wird im Folgenden mit LVÜAC abgekürzt. Sie werden ihr in OOP1 und OOP2 und hoffentlich ihr ganzes Berufsleben lang folgen ;-).

LVÜAC in 5Code

Da LVÜAC die universelle Vorgehensweise beim Programmieren ist, ist es günstig, dass sie von 5Code besonders gut unterstützt wird:

  • Lesen und Verstehen - Aufgaben liegen in editierbarer Form vor
    • Sie können mit den Textmarker-Funktionen wichtige Teile markieren
    • Die Tooltips der Icons geben auch an, mit welchen Tastatur-Kürzeln (shortcuts) selektierter Text formatiert werden kann (Strg-1,2,3,4).
  • Überlegen und Aufschreiben
    • Sie können direkt in die Aufgaben Ihre Überlegungen und Notizen hineinschreiben.
    • Damit man Ihre Notizen auch leicht wiederfindet, markieren Sie sie ebenfalls.
    • Es gibt noch eine weitere Möglichkeit, mit der Sie sich Überlegungen notieren können. Diese Notizen sehen Sie in der Sidebar ganz rechts, wie im Kapitel ChatBot_Notizen erläutert wird.   
  • Codieren
    • Code wird direkt im Anschluss an Ihre Notizen und die konkrete Aufgabenstellung geschrieben

So ist der Zusammenhang immer gegeben:

  • Was SOLL der Code tun - die Aufgabenstellung
  • Wie IST die Funktion Ihres Code tatsächlich - Ihre eigene Lösung 

Dieser Zusammenhang ist immer wichtig, wenn der Code nicht funktioniert. Dann können Sie immer die

  • Anforderung (SOLL) mit Ihrer
  • Umsetzung in Code (IST) vergleichen

Nur einen Teil des Codes ausführen

Wie Sie auch hier sehen, werden so alle Zeilen ausgeführt, die Sie im Code-Rahmen geschrieben haben: Es wird zuerst die Begrüßung angezeigt, danach das Menü.

Bei der Bearbeitung der Aufgaben, beim Testen und im Praktikumstermin kann es jedoch manchmal sinnvoll sein, nur einen Teil des Codes auszuführen, z.B. in diesen Fällen

  • Sie möchten nur die Ausgabe der aktuellen Aufgabe auf der Konsole sehen
  • Sie möchten Fehlerfälle testen, ohne den gesamten Code damit nicht-compilierbar zu machen. 

Nur selektierte Code-Zeilen im Code-Rahmen ausführen

Daher gibt es in 5Code speziell zum Testen die Möglichkeit

  • Code-Zeilen zu selektieren
  • Nur diese Code-Zeilen innerhalb des Code-Rahmens auszuführen

Dazu gehen Sie so vor:

  • Selektieren Sie die gewünschten Code-Zeilen (mit der Maus oder Shift-Cursor-Bewegungen)
  • Öffnen Sie das Kontextmenü (Rechte Maustaste) und wählen Sie die Funktion "Nur selektierten Code im Code-Rahmen ausführen"

Probieren Sie diese Funktion nun aus, z.B. indem Sie

  • zuerst den Code zur Begrüßung alleine ausführen
  • danach den Code für das Menü alleine ausführen. 

Test-Fehlerfälle separat testen

In den hier gestellten Aufgaben kommt es häufig vor, dass Sie nicht nur korrekten Code schreiben sollen, sondern außerdem noch ausprobieren sollen, was im Fall von fehlerhaftem Code passiert. 

Damit dieser absichtlich fehlerhafte Code nicht verhindert, dass das gesamte Lösungs-Dokument unbrauchbar wird, gehen Sie so vor:

  • Sie schreiben den fehlerhaften Code
  • und rahmen ihn mit einem Block-Kommentar ein 

Beispiel:

/* Fehlerfall
    das ist falscher Code
*/

Fügen Sie diesen Fehlerfall nun in Ihren Lösungs-Code ein.

Nun führen sie ausschließlich diesen Fehler-Code aus. Wichtig: selektieren Sie dafür nur die Zeilen mit dem fehlerhaften Code, nicht (!) aber die Kommentarzeilen /* und */! Andernfalls findet der Compiler keinen Fehler, da in einem Kommentar alles stehen darf.

Führen Sie nun noch einmal den gesamten Code-Rahmen aus. Nun sollten sowohl die Begrüßung als auch das Menü auf der Konsole erscheinen. Der Code des Fehlerfalls wurde, wie gewollt, vom Compiler nicht beachtet. 

 

Nutzung der Konsole

Alle Text-Ausgaben erscheinen immer auf der Konsole, im Konsolen-Fenster am unteren Rand des 5Code-Fensters. Was dort erscheinen soll, bestimmen Sie in Ihrem Programm, wie Sie es oben getan haben. Die Ausgaben können Sie in der Konsole nicht editieren.

Sie können den Text in der Konsole, wenn Sie möchten, kopieren und in einem anderen Text oder in einem Code-Kommentar wieder einsetzen. Das ist ganz günstig, um seine Ergebnisse zu dokumentieren.

Wenn die Konsole zu klein ist und nicht alles anzeigt, haben Sie mehrere Möglichkeiten
 
 
 
 

Außerdem:

Wenn Sie die Konsole leeren, d.h. ihren Inhalt löschen möchten, klicken Sie auf das entsprechende Icon am rechten Rand der Konsole
