Prozeduren - Einmaleins

Nun schreiben Sie der Reihe nach einige Prozeduren und rufen sie auf. Dabei nehmen wir das Beispiel des Einmaleins wieder auf, das Sie bereits in einem vorangegangenen Termin bearbeitet hatten.

Sie definieren im Folgenden der Reihe nach einige Prozeduren und testen Sie schrittweise und aufeinander aufbauend - beides auf die Art und Weise, die Sie im letzten Kapitel kennengelernt haben.

Um den Umgang mit Prozeduren und Parametern zu üben, werden die einzelnen Bestandteile der Darstellung in einzelne Prozeduren aufgeteilt. 

Neben der reinen Funktionalität und der korrekten Syntax müssen Sie die weiteren Randbedingungen beachten, die im Coding Style vorgegeben sind.
Speziell für Prozeduren gilt:

- Geben sie den Prozeduren und den Parametern gut beschreibende Namen.  Der Name sollte beschreiben, was die Prozedur tut, in der Form eines Befehls.

- Zu jeder Definition einer Prozedur gehört auch immer die Beschreibung, WAS diese Prozedur tun soll. Das geschieht im javadoc-Format, in dem auch die Parameter mit speziellen Tags beschrieben werden. Die Beschreibung kann kurz sein, muss aber vollständig sein.

Führen Sie dann das .java-File aus. Nun sollten der Titel und die Infozeile angezeigt werden.

 

Spaltenüberschriften anzeigen

Nun brauchen Sie eine Prozedur, die die Spaltenüberschriften für das Einmaleins von 1 bis max auf die Konsole ausgibt. 

Definieren Sie eine Prozedur, die die Zahlen von 1 bis max hintereinander, getrennt mit einem Tabulator-Zeichen in einer Zeile auf die Konsole ausgibt. Hier muss also max als Parameter angegeben und verwendet werden.

Hinweis: Außerdem sollten Sie am Anfang der Überschrift die Zeilennummer berücksichtigen, die am Anfang jeder Zeile stehen soll (siehe nächste Aufgabe). Fügen Sie daher direkt am Anfang der Überschrift ein Tabulator-Zeichen "\t" ein.  

Testen Sie diese Prozedur, indem Sie sie in main() aufrufen. Dafür müssen Sie einen passenden aktuellen Parameter angeben.

Führen Sie dann das .java-File aus. Nun sollten der Titel, die Überschrift und die Infozeile angezeigt werden. 

 

Eine einzelne Zeile anzeigen

Definieren Sie eine Prozedur, die eine einzelne Zeile des Einmaleins von 1 bis max anzeigt. Diese Prozedur bekommt die Zeilennummer und die Zahl der Spalten (d.h. max) als Parameter übergeben. Bei der Ausgabe gilt:

  • Am Anfang der Zeile steht die Zeilennummer, gefolgt von einem Tabulator-Zeichen
  • Die Werte werden der Reihe nach ausgegeben.

Testen Sie diese Prozedur, indem Sie sie in main() aufrufen. Dafür müssen Sie wieder passende aktuelle Parameter angeben. Die einzelne Zeile sollte nach dem Titel, den Spaltenüberschriften und der Infozeile angezeigt werden. 

Dass Sie zum Testen das .java-File immer aufrufen müssen, ist Ihnen inzwischen klar, es wird daher im Folgenden nicht weiter explizit gefordert.

Fehlerfälle testen

Natürlich kann man nicht nur bei der Definition von Prozeduren Fehler machen, sondern auch beim Benutzen, d.h. bei Aufrufen. Einige typische Fehlerfälle sollten Sie kennen, damit Sie diese Fehler erkennen und beheben können. 

Auskommentieren von Fehlercode

Für das Auskommentieren des Fehlercodes haben Sie zwei Möglichkeiten:

  • mit einem Block-Kommentar, wie schon angewendet /* Fehlercode */
    • Das geht natürlich. Hier besteht aber die Möglichkeit, dass man ggf. bereits vorhandene Block-Kommentare damit einschließt. Da Block-Kommentare aber nicht schachtelbar sind, würden sich hierdurch Fehlermeldungen ergeben.
  • mit Zeilenkommentaren // für jede betroffene Zeile
    • Das klappt immer, ohne die o.a. Probleme mit Block-Kommentare
    • Das ist nur etwas lästig, weil man jede einzelne Zeile auskommentieren muss.

Für die letzte Möglichkeit gibt es daher eine eigene Funktion im Code-Editor von 5Code:

  • Selektieren sie die Zeilen, die gemeinsam auskommentiert werden sollen
  • Rufen Sie die Funktion "Selektierten Code ein/auskommentieren" auf
    • Dafür gibt es in der Toolbar ein Icon:

Mit dieser Funktion können Sie

  • sowohl mehrere Zeilen auskommentieren,
  • als auch wieder "einkommentieren", d.h. die Kommentar-// wieder entfernen.
    • Dafür selektieren Sie die auskommentierten Zeilen und klicken das Funktions-Icon.
