Fehlerfall testen

Fehler macht jede*r! Es kommt daher häufig vor, dass Sie einen Fehlerfall testen sollen. Das ist wichtig, damit Sie Fehler sowohl erkennen können, als auch damit umgehen können. Sie sollen also auch Code schreiben, der einen oder mehrere Fehler hat.

Andererseits soll der gesamte Code-Rahmen weiterhin fehlerfrei bleiben, compilierbar sein und auch ausgeführt werden können.

Daher schreiben Sie den fehlerhaften Code innerhalb eines Block-Kommentars

Editieren einer Zeichnung

Klicken Sie mit der Maus auf eine Zeichnung, dann öffnet sich in einem neuen Tab der Editor. Den nutzen Sie wie folgt:

  • Einfügen von neuen Elementen: Ziehen Sie aus dem Menü links ein Element auf die Zeichenfläche (genauso, wie bei der Concept Map).
  • Editieren eines Speicher-Blocks: Doppelklicken Sie auf einen Speicher-Block der auf der Zeichenfläche liegt. Nun können Sie den Namen der zugeordneten Variablen, ihren Wert und ihre Adresse eintragen. 
  • Verbindungen einfügen: Wie bei der Concept Map werden grüne Verbindungs-Punkte angezeigt, wenn Sie mit der Maus über ein Element gehen. Ziehen Sie mit der Maus von diesem Verbindungs-Punkt eine Verbindung zu einem anderen.
