Prozeduren - Handhabung im Code und in der IDE 5Code

In diesem Kapitel lernen sie das "Handwerkszeug" kennen, mit dem Sie Prozeduren und Funktionen im Code schreiben und verwenden.

Tipp: Sie können sich mit einem Rechtsklick auf den Tab eines Dokuments seine Überschriften anzeigen lassen und mit einem Klick direkt dorthin springen. Das geht auch bei Code-Files, dort springen Sie zu den Definitionen.

Unterscheidung Definition - Aufruf

Bisher haben Sie alles, was Sie als Code geschrieben haben, in einem bereitgestellten Code-Rahmen innerhalb der Methode main() geschrieben und auch sofort auch direkt ausgeführt. Wenn Sie nun Code in Prozeduren und Funktionen schreiben, können Sie diesen  Service leider nicht mehr in Anspruch nehmen, da Sie nun unterscheiden müssen zwischen:

  • der Definition der Prozedur - es wird einmal codiert, was passieren soll
    • Aber: nur dadurch, dass eine Prozedur definiert wird, wird ihr Code noch nicht ausgeführt
  • dem Aufruf der Prozedur - es kann beliebig oft bestimmt werden, dass es passieren soll
    • Denn: erst durch den expliziten Aufruf einer Prozedur wird ihr Code auch ausgeführt.

Diese müssen an unterschiedlichen Stellen im Code positioniert werden, wie in der Vorlesung bereits dargestellt wurde. Aber auch dabei unterstützt Sie 5Code mit den neuen Aufgaben-Typen >D und >M, die Sie im Folgenden kennenlernen.

 

Codieren im .java-File

Nun, da Sie mit Prozeduren und Funktionen arbeiten, funktioniert das Erstellen Ihrer Lösungen ein kleines bißchen anders.

Bisher haben Sie innerhalb eines vorgegebenen Code-Rahmens Ihre Lösungen programmiert. Die Definition der "Klasse" und der Prozedur main() konnten Sie dabei nicht ändern, um keine unnötigen Fehler zu machen.

Ab sofort schreiben Sie Ihren Lösungs-Code in "normale" .java-Dateien, in einer Klassen-Definition, und nicht mehr in einem vorgegebenen, nicht editierbaren Code-Rahmen. 

Zu einem späteren Zeitpunkt lernen Sie mehr darüber, was Klassen sind und was diese tun. Für jetzt reicht es, zu wissen, dass man darin auszuführenden Code, Prozeduren und Funktionen schreiben kann.

Öffnen und betrachten Sie die Datei Aufgabe_05.java, welche bereits in diesem Projekt bereitgestellt wurde.

Sie finden die Datei links in der Navigation, wo Sie diese durch Klick öffnen können. Gleichzeitig entfällt der Link zum Code-Rahmen (in der Toolbar oben rechts).

 

Ausführbarer Code in der Prozedur main()

Auszuführenden Code haben Sie bisher auch schon in der speziellen Prozedur public static void main() codiert. Diese Prozedur muss in jedem startbaren Java-Programm vorhanden sein, da sie als Startpunkt jeder Java-Anwendung dient.

Da diese Prozedur main() nicht mehr automatisch generiert wird, müssen Sie sie ggf. selber schreiben. Im Beispiel-File Aufgabe_05.java ist sie für den Anfang aber schon vorhanden. Die Aufgaben-Markierung hat sich daher leicht verändert, sie hat nun die Markierung >M für main().

Schreiben Sie Code, der "Beginn Termin 05" auf die Konsole ausgibt, in main().

Führen Sie den Code in der Klasse Aufgabe_05 aus.

Nun sollte Beginn Termin 05 auf der Konsole ausgegeben werden. 

 

Definition von Prozeduren und Funktionen

Nun definieren Sie eigene Prozeduren und Funktionen. 

Zur Definition gehört auch immer eine Beschreibung. Wie man Prozeduren und Funktionen, u.a. im javadoc-Format, beschreibt, haben Sie bereits im erweiterten Coding Style im Kapitel Beschreibung von Prozeduren und Funktionen gefunden.

Prozeduren und Funktionen werden innerhalb einer" Klasse" definiert, zwischen der ersten { und der letzten } (d.h. im Body der Klasse). Ein Beispiel in Aufgabe_05.java ist die bereits vorhandene Prozedur public static void main().

Prozeduren und Funktionen werden nicht verschachtelt definiert!
 

Damit der Code Ihrer Definition nicht in main() landet, gibt es dafür einen eigenen Aufgabentyp mit der Markierung >D für Definition. 

Wenn Sie auf die >D-Aufgaben-Markierung klicken, wird - wie Sie das gewohnt sind - automatisch an der richtigen Stelle in der Java-Datei ein Kommentar mit der zugehörigen Überschrift generiert. An diese Stelle schreiben Sie dann die Definition Ihrer Prozedur.

Prozedur static void begruessung()

Definieren Sie eine eigene Prozedur static void begruessung(), ohne Parameter, die einen allgemeinen Begrüßungstext auf die Konsole ausgibt.

Da diese Prozedur weder einen Parameter hat, noch einen Rückgabe-Wert, brauchen diese Tags auch nicht in der javadoc-Beschreibung stehen. 

Hinweis: Vergessen Sie nicht das "Zauberwort" static bei der Definition, sonst können Sie die Prozedur/Funktion später nicht direkt in main() aufrufen.

 

 

Aufruf von Prozeduren und Funktionen

Oben haben Sie die Prozedur definiert, d.h. Sie haben festgelegt, was passieren soll, wenn sie ausgeführt wird. Damit die Anweisungen darin ausgeführt werden, müssen Sie die Prozedur zusätzlich noch aufrufen.

Diesen Aufruf schreiben Sie innerhalb des Codes der Prozedur public static void main(). Sie erinnern sich, darin befindet sich der Code, der beim Starten des Programms (dieses java-Files) ausgeführt wird.

Ergänzen Sie main() um einen Aufruf der Prozedur begruessung().
und führen Sie die Klasse / das .java-File anschließend aus.

Überladen von Prozeduren

Sie können nun eine Begrüßung auf der Konsole ausgeben. Im nächsten Schritt wollen Sie diese um einen Namen und eine Anzahl erweitern. Zu diesem Zweck überladen Sie im Folgenden die Prozedur begruessung().

Was bedeutet "eine Prozedur überladen"?
 
 

 

Definition von überladener Prozedur mit Namen

Definieren Sie eine Prozedur zur Begrüßung einer Person, indem Sie die Prozedur begruessung() so überladen, dass als Parameter ein String für den Namen der Person übergeben wird.

Diese Prozedur hat einen Parameter dieser muss daher in der javadoc-Beschreibung kurz beschrieben werden.

Definition von überladener Prozedur mit Namen und Anzahl

Überladen Sie die Prozedur begruessung() erneut, indem Sie zusätzlich einen int-Parameter ergänzen. Dieser bestimmt, wie oft die Begrüßung einer Person ausgegeben werden soll. Definieren Sie auch diese Prozedur zur mehrmaligen Begrüßung.

Diese Prozedur hat zwei Parameter diese werden daher beide in der javadoc-Beschreibung kurz beschrieben.

Aufruf von überladenen Prozeduren

Zuletzt testen Sie die überladenen Prozeduren, indem Sie sie einzeln in main() aufrufen. Im Aufruf müssen Sie jeweils passende aktuelle Parameter angeben.

 

Spezielle Hilfestellungen in 5Code

Die IDE 5Code stellt Ihnen ein paar Funktionen zur Verfügung, die Ihnen die Arbeit erleichtern. In diesem Kapitel werden einige beschrieben, die bei der Codierung von Prozeduren und Funktionen relevant sind.

Fehlerüberprüfung

Um beim Editieren des Codes diesen auf Korrektheit zu überprüfen, klicken Sie den Kompilieren-Button ("Smiley") links oben in der Toolbar oder nutzen Sie den Shortcut (Strg-K). Damit rufen Sie den Compiler für dieses .java-File auf.

Durch den Kompiliervorgang können Sie den Code auf Fehler überprüfen lassen, ohne das Programm auszuführen. Die Ausgaben finden Sie wie gewohnt unten auf der Konsole.

Zusätzlich gibt es, beim hover über das Icon Hinweise zu möglichen Syntax-Fehlern:

Wird das Smiley / der Kompilieren-Button rot dargestellt, können Sie den Maus-Cursor über das Icon bewegen und erhalten nützliche Fehlerhinweise.

Überprüfen Sie auf diese Art und Weise, ob Ihr bisher geschriebener Code korrekt ist.

 

Aktiver Kommentar /// 

Um die Übersicht in der immer umfangreicher werdenden Ausgabe auf die Konsole zu behalten, hatten Sie bereits den Tipp bekommen, die Überschrift der Aufgabe ebenfalls auf die Konsole auszugeben. Diese Ausgabe wird Ihnen durch das 5Code-Feature "Aktiver Kommentar" sehr erleichtert. 

Der aktive Kommentar wird geschrieben als ///

  • Der Text des Kommentars wird in 5Code beim Lauf des Programms auf der Konsole dargestellt. Damit können Sie gut verfolgen, in welcher Reihenfolge Ihr Code abläuft.
  • Syntaktisch ist das ein normaler Java-Zeilenkommentar, den Sie sowieso schon nutzen, um zu beschreiben, was der Code tun soll.

So haben Sie die Möglichkeit, schnell und einfach Konsolenausgaben zu codieren. Zeilen-Kommentare, die mit /// beginnen, werden intern in 5Code durch einen Aufruf von System.out.println() ersetzt.

Nun erscheinen auch die von 5Code automatisch generierten Kommentare mit den Überschriften bei der Ausgabe auf der Konsole!

Sie haben damit bei vielen Ausgaben auf die Konsole immer den Überblick, welche Ausgaben zu welchem Aufgabenteil gehören. Nutzen Sie dieses Feature auch für eigene aktive Kommentare, um Ihren Code und Ihre Konsolenausgaben - auf einfache Weise - verständlicher und übersichtlicher zu machen.

Das Feature "Aktiver Kommentar" sollte sich beim Öffnen des Projekts freigeschaltet haben. Sie können es in der Feature-Liste finden und aktivieren, falls Sie dies noch nicht getan haben.

Probieren Sie einen eigenen aktiven Kommentar in main() aus. 

Die ///-Ausgabe können Sie in der Menüleiste der Konsole auch aus/einschalten.
