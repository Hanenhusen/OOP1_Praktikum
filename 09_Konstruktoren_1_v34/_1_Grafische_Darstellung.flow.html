Grafik-Konsole

Auf der Grafik-Konsole können Sie einfache grafische Elemente zeichnen, z.B. mit der Klasse Button, die in 5Code verfügbar ist. 

Das Feature Grafik-Konsole wurde bereits beim Öffnen dieses Projekts freigeschaltet. 
 

Um das Spiel attraktiver gestalten zu können, soll eine Karte nicht auf der Text-Konsole ausgegeben werden, sondern auf der Grafik-Konsole.

Für die Darstellung jeder Karte verwenden wir nun ein Objekt der Klasse Button. Die Klasse Button ist eine in 5Code mitgelieferte Klasse. Ein Button stellt einen Text an einer bestimmten Position vor einer Hintergrundfarbe auf der Grafik-Konsole dar.

Auf einen Button klicken -> Reaktion

Ein Button ist natürlich dafür gedacht, dass man darauf klickt, und dass dann etwas passiert, d.h. dass es eine Reaktion gibt. In "richtigen" Grafik-Libraries ist das sehr leistungsfähig aber auch recht kompliziert handzuhaben. Damit es für Sie nicht zu kompliziert wird, und damit Sie sich auf die Grundlagen der OOP konzentrieren können, bietet die Grafik-Konsole einen sehr einfachen Mechanismus, um den Klick auf einen Button auszuwerten:

Jeder Klick auf einen Button simuliert als Reaktion eine Eingabe auf der Tastatur.

Fehlerbehebung

Meine Buttons werden nicht (korrekt) angezeigt, was sollte ich nochmal überprüfen?

  • Stellen Sie sicher, dass Sie die aktuelle Version von 5Code nutzen. Manchmal werden im Verlauf des Semesters Änderungen an der Grafik-Konsole vorgenommen, die zur Umsetzung der Aufgaben nötig sind.
  • Achten Sie beim Setzen der Farbe eines Buttons darauf, dass Sie eine gültige Farbbezeichnung (siehe oben)in Form der vordefinierten Farben der Grafik-Konsole verwenden, z.B. Grafik.BLUE. , ein  Ausdruck mit dem Datentyp String.
    • "Grafik.BLUE" , d.h. als String-Literal ist dagegen falsch!
  • Denken Sie stets daran, nicht nur die Instanz-Variablen Ihrer Objekte zu verändern. Soll etwas in der Grafik-Konsole aktualisiert werden, müssen Sie zusätzlich die entsprechenden Methoden der Button-Objekte aufrufen (z.B. moveTo(), setFillcolor()).

Löschen eines Grafikobjekts

Jedes Grafikobjekt kann nachträglich auch wieder gelöscht werden.

Code:

/// Δe_maqrhu Δ 1.1.2 Grafik-Konsole löschen
// Grafik-Konsole löschen
GrafikKonsole.clearGrafik();

/// Δe_l0mdim Δ 1.2.2 Einen einfachen Button erzeugen
// Einfachste Form des Buttons: Nur Position (x, y) und Text angeben:
// Die Position gibt dabei die Position der oberen linken Ecke des Buttons an
// Die Koordinaten werden dabei in Pixeln gezählt
// Dabei ist die Position (0, 0) oben links auf der Konsole
GrafikKonsole.clearGrafik();
Button einfachButton = new Button (40, 20, "Einfach");


/// Δe_jjse86 Δ 1.2.3 Button mit überladenen Constructoren erzeugen
// Normaler Button: Position (x,y), Hintergrund-Farbe,
// Text, Text-Farbe und Reaktion angeben:
GrafikKonsole.clearGrafik();
Button normalButton = new Button (140, 80, Grafik.GREEN,
                                  "Normal", Grafik.BLACK, "Normal geklickt");


/// Δe_ob80tu Δ 1.2.4 Position und Farbe des Buttons setzen
GrafikKonsole.clearGrafik();

// Normaler Button: Position (x,y), Hintergrund-Farbe,
// Text, Text-Farbe und Reaktion angeben:
Button moveButton = new Button (10, 10, Grafik.GREEN,
                                "Move It", Grafik.BLACK, "huhu");
moveButton.moveTo(200, 100);
moveButton.setFillcolor(Grafik.BLUE);


/// Δe_a0p72u Δ 1.2.4 Position und Farbe des Buttons setzen
// erlaubte Farb-Ausdrücke:
// Grafik.BLACK, Grafik.WHITE, Grafik.GREY, Grafik.RED, Grafik.BLUE,
// Grafik.GREEN, Grafik.YELLOW, Grafik.MAGENTA, Grafik.CYAN

/// Δe_yt9bi4 Δ 1.2.5 Eingabe von der Tastatur einlesen
System.out.println("Geben Sie einen Text ein und drücken Sie return");
String eingabe1 = Konsole.eingabeString();  // liest den Text von der Tastatur ein
System.out.println("Dieser String wurde als Eingabe gelesen: " + eingabe1);


/// Δe_vmu0x8 Δ 1.2.6 Auf einen Button klicken
